# Как писать запросы

Harvi Code понимает обычный язык. Тебе не нужны специальные команды или синтаксис для общения с ним. Просто пиши свой запрос обычными словами, как будто говоришь с живым разработчиком.

---

## Стратегии эффективных запросов

Четко формулируй, что ты хочешь от Harvi. Избегай расплывчатых или двусмысленных формулировок.

| Стратегия | Реализация |
|-----------|------------|
| **Будь конкретным** | "Исправь баг в `calculateTotal`, который возвращает неправильный результат" вместо "Исправь код" |
| **Давай контекст** | Используй @ [упоминания контекста](/basic-usage/context-mentions) для ссылок на файлы и код |
| **Разбивай задачи** | Отправляй сложные задачи маленькими управляемыми шагами |
| **Приводи примеры** | Показывай пример кода, если нужен конкретный формат или стиль |

## Типичные ошибки

| НЕ НАДО | НАДО |
|---------|------|
| Расплывчатые запросы | Точно указывай, что нужно сделать |
| Предполагать контекст | Явно ссылайся на файлы и функции |
| Избыток технического жаргона | Используй понятный, прямой язык |
| Несколько несвязанных задач | Отправляй по одному фокусированному запросу |
| Продолжать без проверки | Проверяй код, чтобы убедиться, что все готово |

---

## Советы для лучших результатов

- **Начинай с простого:** Если задача сложная, начни с общего описания, а потом уточняй детали
- **Используй примеры:** Покажи Harvi Code, как должен выглядеть результат
- **Давай обратную связь:** Если результат не тот, что ожидал - скажи об этом и уточни
- **Экспериментируй:** Пробуй разные формулировки, чтобы найти свой стиль общения
