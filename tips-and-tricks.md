Подборка практических советов, которые помогут тебе работать с Harvi Code максимально эффективно.

## Организация интерфейса

Перетащи Harvi Code на [вторую боковую панель](https://code.visualstudio.com/api/ux-guidelines/sidebars#secondary-sidebar), чтобы одновременно видеть Explorer, Search, Source Control и другие панели.

Когда Harvi Code находится на отдельной боковой панели, можешь перетаскивать файлы из Explorer прямо в окно чата (даже несколько сразу). Просто зажми Shift после начала перетаскивания.

## Оптимизация производительности

Если не используешь MCP, отключи его в настройках — это значительно сократит размер системного промпта и ускорит работу.

Для кастомных режимов ограничь типы файлов, которые они могут редактировать — это поможет агенту оставаться в рамках задачи.

## Работа с контекстом

Если столкнулся с ошибкой `input length and max tokens exceed context limit`, можешь:
- Удалить сообщение
- Откатиться к предыдущему чекпоинту
- Переключиться на модель с большим контекстным окном

Будь внимателен с настройкой `Max Tokens` для thinking-моделей. Каждый токен, выделенный на размышления, отнимается от места для истории разговора. Используй высокие значения `Max Tokens` только для режимов Architect и Debug, а для Code держи 16k или меньше.

## Работа с режимами

Если есть реальная вакансия для задачи, которую должен выполнять кастомный режим, попроси Code режим: "Создай кастомный режим на основе вакансии по адресу @[url]"

При использовании Debug режима проси Harvi "начать новую задачу в Debug режиме со всем необходимым контекстом для выяснения X" — так процесс отладки использует своё контекстное окно и не засоряет основную задачу.

## Параллельная работа

Для максимального ускорения работы сделай несколько копий репозитория и запусти Harvi Code на всех параллельно. Конфликты разрешай через git, как при работе с несколькими разработчиками.

## Управление большими файлами

Для работы с большими файлами настрой параметр `File read auto-truncate threshold` в настройках. Он контролирует количество строк, читаемых из файла за раз. Меньшие значения улучшают производительность при работе с очень большими файлами, но могут потребовать больше операций чтения.

## Горячие клавиши

Настрой горячую клавишу для команды `harvi-code.acceptInput`, чтобы принимать предложения или отправлять текст без использования мыши. Идеально для работы с клавиатуры и снижения нагрузки на руки.

## Кастомизация контекста

Если агент забывает важную информацию при сокращении контекста, настрой промпт для сокращения контекста. Можешь указать, какие типы информации критичны для твоего workflow и должны сохраняться.

Эти советы помогут тебе работать с Harvi Code быстрее и эффективнее!
